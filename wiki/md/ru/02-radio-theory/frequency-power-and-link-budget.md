# Частота, мощность и link budget

Дальность радиосвязи определяется не одним параметром мощности. Сигнал проходит цепочку: передатчик → кабель → антенна → пространство → антенна → кабель → приёмник. Каждый элемент добавляет усиление или потери. **Link budget** сводит их в один расчёт и показывает, остаётся ли сигнал выше чувствительности приёмника с достаточным запасом.

## Частота и длина волны

Частота `f` показывает число колебаний в секунду. Длина волны приблизительно равна:

```text
λ = c / f
```

где `c` — скорость света. Для практической оценки:

```text
λ, м ≈ 300 / f, МГц
```

Примеры:

| Частота | Длина волны |
|---:|---:|
| 433 МГц | около 0,69 м |
| 868 МГц | около 0,35 м |
| 915 МГц | около 0,33 м |
| 2,4 ГГц | около 0,125 м |

Длина волны влияет на физический размер антенны, дифракцию, потери в кабеле и проникновение через препятствия. Более низкая частота не гарантирует большую дальность, но при равных условиях обычно легче огибает препятствия и требует более крупной антенны.

## dB и dBm

**Децибел** (`dB`) — относительная логарифмическая величина. Он удобен тем, что усиления и потери складываются.

- удвоение мощности ≈ `+3 dB`;
- уменьшение мощности вдвое ≈ `−3 dB`;
- увеличение в десять раз = `+10 dB`;
- уменьшение в десять раз = `−10 dB`.

**dBm** — абсолютная мощность относительно 1 мВт:

```text
P(dBm) = 10 · log10(P(mW))
```

Ориентиры:

| dBm | Мощность |
|---:|---:|
| 0 dBm | 1 мВт |
| 10 dBm | 10 мВт |
| 14 dBm | около 25 мВт |
| 20 dBm | 100 мВт |
| 22 dBm | около 158 мВт |
| 30 dBm | 1 Вт |

Значение `set tx 22` относится к настройке LoRa-чипа. Если плата содержит внешний PA, реальная мощность на антенном разъёме может быть выше. И наоборот, несовершенный matching и защитные цепи могут уменьшить её.

## dBi, dBd, ERP и EIRP

**dBi** описывает усиление антенны относительно идеального изотропного излучателя. **dBd** — относительно полуволнового диполя. Приблизительное соотношение:

```text
0 dBd ≈ 2,15 dBi
```

**EIRP** учитывает мощность на выходе передатчика, потери фидера и усиление антенны:

```text
EIRP = TX power − cable loss − connector loss + antenna gain
```

Например:

```text
14 dBm − 2 dB + 5 dBi = 17 dBm EIRP
```

Регуляторные ограничения часто задаются именно для EIRP или ERP. Поэтому установка антенны с высоким gain может потребовать снижения `tx`.

## Чувствительность приёмника

Чувствительность — минимальный уровень сигнала, при котором приёмник достигает заданной вероятности ошибок. Для LoRa она зависит от:

- bandwidth;
- spreading factor;
- coding rate;
- реализации радиочипа;
- температуры и частотной ошибки;
- качества платы и питания;
- выбранного критерия PER.

Нельзя использовать одно число чувствительности для всех LoRa-профилей. Узкая BW и высокий SF обычно улучшают чувствительность, но увеличивают airtime.

## Базовый link budget

Максимально допустимые потери линии можно оценить так:

```text
Lmax = Ptx + Gtx − Ltx + Grx − Lrx − Sensitivity
```

где:

- `Ptx` — мощность передатчика в dBm;
- `Gtx`, `Grx` — усиление антенн в dBi;
- `Ltx`, `Lrx` — потери кабелей и соединений в dB;
- `Sensitivity` — отрицательное значение чувствительности в dBm.

Пример:

```text
Ptx          = 14 dBm
Gtx          =  2 dBi
Ltx          =  2 dB
Grx          =  2 dBi
Lrx          =  1 dB
Sensitivity  = -130 dBm

Lmax = 14 + 2 - 2 + 2 - 1 - (-130) = 145 dB
```

145 dB — теоретический предел до чувствительности. Рабочую линию нельзя проектировать на этом пределе.

## Fade margin

**Fade margin** — запас на изменения среды. Для стационарной линии с хорошей видимостью может потребоваться один запас, для леса, города или мобильного узла — значительно больший.

Если из предыдущих 145 dB вычесть 15 dB запаса, допустимые расчётные потери становятся 130 dB. Этот запас покрывает часть эффектов:

- дождь и мокрую растительность;
- изменение положения антенны;
- многолучёвость;
- колебания noise floor;
- температурный уход частоты;
- старение и влажность соединений.

Link margin можно измерить как разницу между фактическим уровнем принятого сигнала и порогом устойчивой работы. Но для LoRa полезнее оценивать совместно RSSI, SNR и PDR, а не только RSSI.

## Free-space path loss

Потери в свободном пространстве:

```text
FSPL(dB) = 32,44 + 20·log10(dкм) + 20·log10(fМГц)
```

Формула предполагает свободное пространство, согласованные антенны и отсутствие препятствий. Она не учитывает рельеф, зону Френеля, здания и шум.

Для одной и той же дистанции повышение частоты увеличивает FSPL. Но реальная линия может вести себя иначе из-за размеров антенн, разрешённой мощности и локальных помех.

## Почему увеличение мощности часто не решает проблему

Дополнительные `+3 dB` означают лишь удвоение мощности. Если линия потеряла 20 dB из-за стены, леса или неправильной поляризации, увеличение с 14 до 17 dBm почти ничего не изменит.

Более эффективные действия:

- поднять антенну;
- освободить зону Френеля;
- убрать длинный плохой кабель;
- исправить поляризацию;
- снизить локальные помехи;
- установить repeater в точке с реальной радиовидимостью;
- выбрать более подходящий LoRa-профиль.

## Двунаправленность линии

MeshCore требует не только доставки исходного пакета, но и часто обратного path/ACK. Линия может быть асимметричной из-за:

- разной мощности плат;
- разных антенн и кабелей;
- внешнего PA только на одной стороне;
- различного noise floor;
- разной высоты;
- перегрузки приёмника сильным локальным передатчиком.

Если A хорошо слышен B, это не доказывает, что B хорошо слышен A. Trace и двусторонняя статистика нужны для проверки обоих направлений.

## Практический расчёт

Для каждой критической связи составляют таблицу:

| Элемент | Значение |
|---|---:|
| TX power на разъёме | ... dBm |
| Потеря TX-кабеля | ... dB |
| Усиление TX-антенны | ... dBi |
| Расчётные path loss | ... dB |
| Усиление RX-антенны | ... dBi |
| Потеря RX-кабеля | ... dB |
| Ожидаемый RSSI | ... dBm |
| Чувствительность профиля | ... dBm |
| Запас | ... dB |

Затем расчёт проверяют реальными пакетами, а не непрерывным несущим сигналом: важны PDR, ACK и стабильность пути в разное время суток.
